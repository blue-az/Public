Purpose

Bulkhead τ is the public release line for a system that does not try to make AI smarter. It tries to make AI governable. The intelligence sits inside a watertight shell, bounded above by a PBC-compatible Charter layer and below by a GT-KB-derived Logbook layer. Inputs arrive as fuel; outputs leave as repeatable insights; nothing leaks.

Where the rest of the AI tooling space is building thin wrappers around language models — thousands of rafts, fast to build and quick to capsize — Bulkhead τ is the engineered alternative: a deterministic substrate that survives contact with real domains.

The Three-Tier Seal

Upper Boundary — PBC

The Charter. Product Behavior Contract documents (.pbc.md) define mission, scope, and routing constraints. This layer is upstream-aligned with PBC's provider-neutral anchoring model, so external evidence can attach to stable behaviors without making PBC depend on any one ledger.

Core — Bulkhead τ

The engine room. Intakes data, executes physics via deterministic substrates, and delivers validated thrust. In Release 0.6.1, the engine is being wired to consume live evidence from the fork-maintained Logbook before treating Charter behaviors as authoritative.

The Evidence-Backed Trust Loop

We have transitioned from a linear lifecycle to an evidence-backed trust loop. This resolves the "overloaded verified" problem by separating historical governance (the Verified milestone) from current evidence (the Passing trust state).

  • PBC remains authored, stable, and provider-neutral.
  • The GT-KB-derived fork changes trust state dynamically as evidence changes (passing, failing, stale, disputed, or blocked).
  • Bulkhead τ provides the downstream defense, gating runtime authority on that live trust state.

Provenance note: the PBC anchoring path is upstream-aligned through merged provider-neutral guidance; the GT-KB trust-state layer is currently maintained in the Bulkhead τ fork.
