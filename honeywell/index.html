This site was designed using Project Phoenix principles

  • Deterministic first: all dashboard claims map back to SQLite-backed source tables.
  • Separation of concerns: data pulls, DB build, tools, and UI are kept independent.
  • Factual vs advisory boundary: hypotheses are separate from verified operational facts.
  • Human-in-the-loop governance: readiness gates and scoring prevent ungrounded conclusions.
