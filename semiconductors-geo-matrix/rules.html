V1–V4 Core Analysis Rules

  • All share queries must specify market and segment.
  • Valid markets: CPU, GPU, Foundry. Valid segments: Client, DataCenter, Discrete, Wafer.
  • Share data is snapshot-based (2024-12-31 or 2025-12-31) — not time-series.
  • Use SearchCompanies before other tools if company name is ambiguous.
  • Ticker aliases are normalized: INTEL → INTC, NVIDIA → NVDA, TSMC → TSM.
  • Concentration queries return HHI-style metrics — cite the segment and date.
  • Financial metrics from SEC company facts; do not extrapolate beyond reported periods.
  • Price performance comparisons require a benchmark ticker (default: SOXX or SPY).

V5–V6 Catalyst + SOP Rules

  • Event queries require company and date range; event_type is optional.
  • Price impact windows default to ±5 trading days around the event date.
  • WriteCatalystNote must precede VerifyCatalystNote — no verify without write.
  • Notes are persisted to data/catalyst_notes/ — do not fabricate file paths.
  • RunCatalystSOP requires: company, event_date, and thesis statement.
  • VerifyCatalystSOP audits all output artifacts for completeness.
  • ListCatalystSOPRuns returns run history — use run_id for verification.

V7–V8 Visualization Rules

  • PNG outputs are written to data/visualizations/ before being reported.
  • Interactive HTML outputs are written to data/visualizations_html/.
  • Dashboard assembly tools require at least one primary visualization to be complete first.
  • Research packs bundle all chart types into a single index file — do not link non-existent files.
  • BuildInteractiveSemiconductorBriefing is the V8 capstone — requires market + ticker.
  • All chart titles must include market segment and snapshot date.
